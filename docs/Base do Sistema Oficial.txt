# 1. 🆔 Identidade do Projeto

**Nome:**
CVF Insight – Plataforma de Diagnóstico Cultural

**Objetivo:**
Desenvolver um sistema web de diagnóstico de cultura organizacional baseado no modelo CVF (Competing Values Framework), com foco em gerar insights acionáveis para empresas.

**Características do sistema:**

* Sistema em produção com dados reais
* Foco em confiabilidade e estabilidade
* Evolução incremental e segura
* Usuário final não técnico
* Ênfase em anonimato e clareza

---

# 2. 📊 Fase Atual e Regra Crítica

**Objetivos da fase:**

* Coleta confiável de dados
* Prevenção de duplicidade

# 3. 🧱 Arquitetura do Sistema

**Stack oficial:**

* Backend: Flask
* Banco de dados: PostgreSQL
* Deploy: Render
* Frontend: HTML + Jinja
* Interatividade: JavaScript

**Princípios arquiteturais:**

* Manter estrutura atual (evitar refatorações desnecessárias)
* Evitar duplicação de lógica
* Integrar novas funcionalidades ao fluxo existente
* Priorizar simplicidade e previsibilidade

---

# 4. ⚙️ Regras de Governança

**Regras obrigatórias:**

* Não alterar arquitetura sem justificativa forte
* Não duplicar lógica existente
* Sempre integrar ao sistema atual
* Sempre indicar onde alterar (arquivo/camada)
* Priorizar baixo risco
* Considerar que o sistema está em produção

**Segurança e estabilidade:**

* Evitar quebra de funcionalidades existentes
* Preservar anonimato do respondente
* Validar impacto antes de implementar
* Preferir mudanças incrementais

**Foco no usuário:**

* Linguagem simples (usuária leiga)
* Passo a passo obrigatório
* Código completo quando necessário
* Explicação de onde salvar e como testar
